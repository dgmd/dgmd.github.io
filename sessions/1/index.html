The next three weeks

Three weeks isn't long to build something. As those who took the semester-long version of this course can attest, we felt rushed with more than fifteen weeks. To make the most of our time, try to immerse yourself in actually programming and working on your project(s). That can look different for different folks, but here are a few, good litmus tests…

  1. brainstorming and publishing project inspirations, ideas, and refinements on your Github pages site every day
  2. writing code—for your own projects or extensions and exercises from class—and pushing it to Github every day
  3. meeting weekly milestones for your personal project, including:
    • Week 1 — Finish a description, specification, and timeline for your final project
    • Week 2 — Create a project demo, including description of what's left to finish for your final project
    • Week 3 — Finish a first version of your final project

Setting Up Your Personal Computer To Code

a-story-in-pomes

To get you thinking about what is possible with these digital tools and what it looks like to design something for another person, we will be looking at this sample project, presented as a gift last year. To look at its insides you will need to:

  1. Fork the the git repository by clicking "Fork" in the upper right of this page.
  2. Now, clone the project by typing git clone https://github.com/YOUR-USERNAME/a-story-in-pomes.git at your terminal.
  3. Change directory into the repository by typing cd a-story-in-pomes at your terminal.
  4. From inside the directory, run git checkout gh-pages to switch to the active branch of the project.
  5. Run python -m SimpleHTTPServer. You should now be able to go to localhost:8000 in Google Chrome to view the website.
  6. Now, if you make changes to the project, you will see them reflected when you refresh this local page, but you will not effect the online version of the project. Try this out by opening the files you downloaded and changing one of the files.
